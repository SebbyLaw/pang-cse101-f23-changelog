Objectives:

Develop a binary search tree (BST) from a sorted array and implement various traversal methods along with insertion and deletion operations.

Description:

In this assignment, you will implement a binary search tree (BST) and develop an application to manage and manipulate the tree.

Details:

  • Your application will begin by reading the number of elements in a sorted array and the sorted array itself from stdin. These will be used to generate the initial binary search tree.
  • The first line read from stdin will represent the number of elements in the sorted array, and the second line will contain the elements of the sorted array itself.
  • After the tree has been initialized, your application should support the following functionalities, executed according to the index of commands from stdin:
    1. Insert a Node: Add a new node with a specified value to the BST.
    2. Delete a Node: Remove a node with a specified value from the BST.
    3. Find a Node: Search for a node with a specified value and return its details.
    4. Inorder Traversal: Traverse and display the BST in an inorder manner.
    5. Preorder Traversal: Traverse and display the BST in a preorder manner.
    6. Postorder Traversal: Traverse and display the BST in a postorder manner.
    7. Exit: Terminate the application.

Files for this program:

  • Makefile: Makefile for compiling your code.
  • BST.h: Header file containing the structure of the BST and the prototypes of the functions you need to implement.
  • BST.c: This is where you implement your binary search tree and its functionalities.
  • hwk4.c: This is the main application file where you will handle user inputs and call the corresponding BST functions.

Input and Output Files:

  • test1.in: This file contains the sorted array followed by commands for stdin. The first line specifies the number of elements, and the second line lists the elements themselves.
  • test1.out: This is the expected standard output file based on the operations performed in test1.in.
  • test2.in: Similar to test1.in, this file contains a sorted array and a set of commands.
  • test2.out: This is the expected standard output file based on the operations performed in test2.in.
